Allowed functions - 42 webserv project

Introduction

This guide provides a brief explanation of C/C++ programming functions allowed in 42-webserv project. It covers functions related to process manipulation, socket management, file and directory I/O, signal handling, and more.

Program Execution

  • execve: Replaces the current process image with a new program.

File Descriptor Management

  • dup and dup2: Duplicates an existing file descriptor.
  • pipe: Creates a communication channel (pipe) between processes.

Error Handling

  • strerror: Converts an error code into a human-readable error message.
  • gai_strerror: Converts an address resolution error code into an error message.

Error Variables

  • errno: A global variable that stores the error code after an error occurs.

Processes

  • fork: Creates a new child process from the parent process.
  • waitpid: Waits for the termination of a child process.
  • kill: Sends a signal to a process.
  • signal: Defines custom signal handling.

Socket Handling

  • socketpair: Creates a pair of local domain sockets.
  • htons, htonl: Converts port and address values to network byte order.
  • ntohs, ntohl: Converts port and address values from network byte order to host byte order.
  • select, poll: Waits for I/O events on multiple file descriptors.
  • epoll (epoll_create, epoll_ctl, epoll_wait): Efficiently manages I/O events using the `epoll` interface.
  • kqueue (kqueue, kevent): Provides similar functionality to `epoll` on BSD systems.

Conclusion

This guide provides an overview of common functions used in WebServ 42 Project.
